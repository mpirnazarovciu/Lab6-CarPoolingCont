Car Pooling Continuation

Develop a Java application, which allows company to organize Car pooling system. There are drivers inside of company which helps employees to reach destination. Car pooling is a system which allows to organize this process and book driver beforehand. All the classes must be in the package "uz.muu".


R1 - Drivers Definition

The system works through the facade class CarPooling. 

The method createDriver() accepts the driver name and creates a Driver object. The class Driver provides the method getName(), which returns the Driver’s name.

The method getDrivers() returns a Collections of Drivers. The method getDriver() receives the drivers name as an input and returns the corresponding Driver object. In case if driver with corresponding name does not exist, exception NoSuchDriver is thrown.

By default drivers work from 9:00 till 18:00. Drivers can choose working hours and employees can book in this timing range. The method setDriverWorkingHours() is used for that.


R2 - Employee

Employees of an organization are added by administrator. The method addEmployee() is used to add employee by means of ID, full name and position. In case if employee with corresponding ID exist, exception EmployeeAlreadyExist is thrown. The class Employee offers the method getName(), which returns the Employees full name.

To obtain a specific Employee it is possible to use the method getEmployee(), which receives the ID of the Employee as a parameter. In case if employee with corresponding ID does not exist, exception NoSuchEmployee is thrown.



R3 - Driver booking

The drivers could be booked by means of employee ID, driving starting time(e.g. 14:00) and total duration (1hr or 2hr). This operation could be done through the method bookDriver(). Method returns driver which is available at that time and makes driver unavailable for that time. In case if there is no driver in that timing slot, exception NoDriverAvailableForBooking is thrown.

Only employees with position "CEO", "CTO" and "Manager" can book the driver. In case if employee has different position, exception BookingDriverNotAllowed is thrown.

The method availableDriver() which takes as an input: driving starting time(e.g. 14:00) and total duration (1hr or 2hr) and returns an object of type Driver, which is available at that time. In case if there is no driver in that timing slot, exception NoDriverAvailableForBooking is thrown.



R4  - Statistics

Given a Drivers name it is possible to get the drivers availability for all day by using the method getDriverTimetable(), which gets the corresponding driver name as an input. 

Given an employee ID it is possible to know how many times he/she booked drivers by using the method getNumberOfBookingsByEmployee().

Given an employee position it is possible to know how many times employees with given position booked drivers by using the method getNumberOfBookingsByEmployeePosition().

The method printAllBookings() prints Collection of number of bookings by all employees and number of bookings sorted in descending way.
